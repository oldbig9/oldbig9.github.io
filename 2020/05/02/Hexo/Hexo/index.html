Hexo+Gitee搭建博客系统

发表于 2020-05-02 更新于 2020-05-22 分类于 Hexo
本文字数： 9.9k 阅读时长 ≈ 9 分钟
做个博客记记笔记用，权当知识积累的一种手段吧，脑子不好使，记不住，写下来方便查阅，一开始打算用 Hexo+Github 的，但是搭建完成后发现 Github 这玩意访问速度太慢，所以就改成码云了，但是免费版码云有个缺点就是每次部署后都需要手动更新，不然看不到最新的内容。

以下内容记录搭建过程以及遇到的坑:

一、安装 Hexo

  1. 安装脚手架

    npm install -g hexo-cli

  2. 创建博客项目

    hexo init project_name

  3. 创建页面

    hexo new page page_name

    例：创建 tags 页面

  4. 创建文章

    hexo new post demo
    生成的 Markdown 文档在 source/_post/demo.md,然后就可以编辑文档了,例:

  5. 本地查看

    hexo server

    然后就可以在浏览器查看了 localhost:4000

  6. 部署到码云
    部署到码云或 Github 需要安装一个插件

  2. 代码托管

    计划部署在码云上，源码也打算托管在码云上，hexo 源码一个仓库，采用子模块方式管理主题，主题仓库 fork 自 theme-next/hexo-theme-next，这样可以更方便的更新主题，但是 next 主题仓库是托管在 github 上的，码云不能直接 fork github 上的仓库，所以还是把代码都托管在 github 上吧，部署也可以部署在 github 上，毕竟不用手动更新部署
